Itu TicketBAI FS

Module that adapts the ticketbai modules (l10n_es_ticketbai_api and l10n_es_ticketbai) available at https://github.com/OCA/l10n-spain/ so that you can create and rectify simplified invoices.

Instalation:

That module requiere this extra modules:
  • account
  • itu_log
  • l10n_es_ticketbai
  • itu_original_invoice_data
  • l10n_es_ticketbai_api.

Configuration:

  • By default the module is configured with a new Journal for simplified invoices using the FS code and with a limit for simplified invoice of 3000€.
  • To configure a customer as anonymous:
    • Go to Contacts > Contact_Name > AEAT tab >
    • Mark the customer as Anonymous (AEAT - Anonymous Customer)
    • Select as Country (Spain) and leave VAT/NIF/CIF blank.
    • Select as Fiscal Position "National Regime".
  • The journal for simplified invoices can be configured in Configuration > General Settings > Companies > Company > Simplified Invoice tab.
  • The limit for simplified invoices can be configured in Configuration > General Settings > Companies > Company > Company > Simplified Invoice tab.

Usage:

Module that adapts the ticketbai modules (l10n_es_ticketbai_api and l10n_es_ticketbai) that you can find in https://github.com/OCA/l10n-spain/ so you can create and rectify simplified invoices from the Invoicing/Accounting screen (Account.move) and send them automatically to TBAI servers.

  • If we have marked the client as Anonymous (AEAT - Anonymous Client) and the client does not have a DNI or CIF it will create a simplified invoice.
  • Create an accounting journal (account.journal) for simplified invoices, this way they will have their own sequence and control.
  • Check that the amount of the simplified invoice does not exceed the legal limit. By default it is 3000€.
  • It can be used with ticketbai Disabled or enabled.
  • Adapts the file that is sent to ticketbai to indicate that it is a simplified invoice or a rectification of a simplified invoice.

Known issues:

The module gives error when creating a simplified invoice if you have not configured the sequence of "normal" invoices or if you have to regenerate it, for example when changing month or year. You can solve this problem by simply generating a "normal" invoice.< /p>

Odoo 14 ITU Sequence by YEAR

Módulo que adapta la secuencia de Facturas a código/Año/Numero Factura:

Instalación:

Este módulo requiere de los módulos:
  • account
  • itu_log

Configuración:

Crea un checkbox en los Diarios de contabilidad (account.journal) para activar o desactivar la secuencia usando solo el año.

Uso:

Al Crear o Regenerar una secuencia en vez de comenzar con CODIGO / AÑO / MES / Nº FACTURA se crea uno CODIGO / AÑO / Nº FACTURA.

Nota: Al crear una entrada en un diario la secuencia se coge de la última entrada, por lo tanto si el Diario ya tenía entradas usando la secuencia CODIGO / AÑO / MES / N.º FACTURA no cambiara hasta que toque regenerarla, por ejemplo con el cambio de mes.

Esto se puede solucionar creando un diario contable nuevo
