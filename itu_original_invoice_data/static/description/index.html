ITU ORIGINAL INVOICE DATA

Saves Company and Customer data on invoice confirmation moment and display original data when you reprint invoice.

Instalation:

That module requiere this extra modules:
  • account
  • itu_log

Configuration:

  • Automaticaly works with basic designs:
    • Light
    • Boxed
    • Clean
    • Background
  • You can easily implement in your custom desings

Usage:

The module save automaticaly partner and company data

Shows a message on the invoice when original data has been modified

Known issues:

The module gives error when creating a simplified invoice if you have not configured the sequence of "normal" invoices or if you have to regenerate it, for example when changing month or year. You can solve this problem by simply generating a "normal" invoice.

Itu TicketBAI FS

Módulo que adapta los módulos de TicketBai (l10n_es_ticketbai_api y l10n_es_ticketbai) que puedes encontrar en https://github.com/OCA/l10n-spain/ para que puedas crear y rectificar facturas simplificadas desde la pantalla de Facturación/Contabilidad (Account.move) y que se envíen automáticamente a los servidores de TBAI.

  • Si tenemos marcado al cliente como Anónimo (AEAT - Cliente anónimo) y el cliente no tiene DNI o CIF creara una factura simplificada.
  • Crea un Diario de contabilidad (account.journal) para facturas simplificadas, de esta manera tendrán su propia secuencia y control.
  • Controla que el importe de la factura simplificada no sea superior al limite legal. Por defecto son 3000€.
  • Se puede usar con TicketBAI Deshabilitado o habilitado.
  • Adapta el archivo que se envía a TicketBAI para indicar que es una factura simplificada o una rectificación de una factura simplificada.

Instalación:

Este módulo requiere de los módulos:
  • account
  • itu_log
  • l10n_es_ticketbai
  • itu_original_invoice_data
  • l10n_es_ticketbai_api.

Configuración:

  • Por defecto el módulo se configura con un Diario nuevo para facturas simplificadas usando el código FS y con un límite para factura simplificada de 3000€.
  • Para configurar un cliente como anónimo:
    • Ir a Contactos > Nombre_Contacto > Pestaña AEAT >
    • Marcar al cliente como Anónimo (AEAT - Cliente anónimo)
    • Seleccionar como País (España) y Dejar el IVA/NIF/CIF en blanco.
    • Seleccionar como Posición fiscal "Régimen Nacional".
  • El diario para facturas simplificadas puede configurarse en Configuración > Ajustes generales > empresas > Empresa > Pestaña Factura simplificada.
  • El límite para las facturas simplificadas puede configurarse en Configuración > Ajustes generales > empresas > Empresa > Pestaña Factura simplificada.

Uso:

Al crear una factura se comprueba si el cliente es anónimo y si tiene DNI/CIF. En caso de que este marcado como anónimo y no tenga DNI/CIF se crea una factura simplificada en vez de una normal.

Problemas Conocidos:

El módulo da error al crear una factura simplificada si no tienes configurada la secuencia de facturas "normales" o si toca regenerarla, por ejemplo al cambiar de mes o año. Puedes solucionar este inconveniente simplemente generando una factura "normal".

ITU TicketBAI FS

Faktura erraztuak Fakturazio/kontabilitate lehiotik (Account.move) egin ahal izateko TicketBai-eko moduloak (l10n_es_ticketbai_api y l10n_es_ticketbai) egokitzen dituen moduloa. TicketBaieko moduluak https://github.com/OCA/l10n-spain/ -en lortu ditzazkezu.

  • Bezeroa anonimo bezala markatzen badegu (AEAT - Cliente anónimo) eta bezeroak NAN/IZF ez badu automatikoki faktura erraztu bat sortzen du
  • Faktura erraztuentzako kontabilitate liburu berri bat sortzen du (account.journal), bere sekuentzia eta egiaztapena errazteko.
  • Faktura erraztuaren totala muga legala baiano handiago ez izatea egiaztatzen du.
  • TicketBai aktibatuta edo desaktibatuta-rekin erabili daiteke.
  • TicketBai aktibatuta izanda faktura erraztuak automatikoki bildaltzen dira.
  • TicketBai-eko fitxategia moldatzen du faktura erraztu bat ala faktura erraztu rektifikatzaile bat dela adierazteko.

Instalazioa:

Modulu Honek beste hauek behar ditu:
  • account
  • itu_log
  • l10n_es_ticketbai
  • itu_original_invoice_data
  • l10n_es_ticketbai_api

Ezarpenak:

  • Modulua instalatzerakoan Kontabilitate liburu berri bat sortzen du FS kodea eta 3000€ -ko mugarekin faktura erraztuentzako.
  • Bezero bat anonimo bezala konfiguratzeko:
    • Bezeroetara joan > kontatuaren izena > AEAT fitxa >
    • Bezeroa anonimo bezala markatu (AEAT - Cliente anónimo)
    • Herrialde bezala España aukeratu eta NAN/IFK txurian utzi
    • Pozizio fiscala bezala "Regimen Nacional" aukeratu
  • Faktura erratuerako kontabilitate liburua konfigurazioak > ezarpen orokorrak > empresak > empresa > Faktura erraztua fitxan konfiguratu daiteke
  • Faktura erraztuaren totalen muga legala konfigurazioak > ezarpen orokorrak > empresak > empresa > Faktura erraztua fitxan konfiguratu daiteke

Erabilpena:

Factura berri bat sortzerakoan bezeroa anonimoa den eta NAN/IFK duen egiaztatzen da. Anonimoa eta NAN/IFK ez badu faktura erraztu bar sortzen da automatikoki

Arazo ezagunak:

Faktura normalen sekuentzia sortugabe baduzu edo berritzea tokatzen bada moduloak errorea emanten du faktura erraztu bat sortzerakoan.

Akats hau faktura normal bat sortuta konpontzen da.
